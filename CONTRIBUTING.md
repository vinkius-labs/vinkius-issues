# Contributing to Vinkius Issues

Thank you for your interest in contributing to Vinkius! This repository serves as the central hub for reporting issues, requesting features, and getting support for Vinkius.

## 📋 Before You Start

- **Search first**: Before creating a new issue, please search existing issues to avoid duplicates
- **Use the right template**: Choose the appropriate issue template for your specific need
- **Be respectful**: Follow our [Code of Conduct](CODE_OF_CONDUCT.md) in all interactions

## 🐛 Reporting Bugs

When reporting bugs, please:

1. **Use the Bug Report template** - This ensures we get all the information we need
2. **Provide detailed steps to reproduce** - Help us understand exactly what happened
3. **Include environment information** - OS, browser, Vinkius version, etc.
4. **Add logs and error messages** - Include any relevant error output
5. **Attach screenshots if applicable** - Visual information can be very helpful

### What makes a good bug report?

- **Clear title** - Summarize the issue in the title
- **Detailed description** - Explain what happened and what you expected
- **Reproducible steps** - Step-by-step instructions to reproduce the issue
- **Environment details** - Your setup information
- **Minimal example** - If possible, provide a minimal example that demonstrates the issue

## ✨ Requesting Features

When requesting features, please:

1. **Use the Feature Request template** - This helps us understand your needs
2. **Explain the problem** - What problem does this feature solve?
3. **Describe your solution** - What would you like to see implemented?
4. **Consider alternatives** - What other solutions have you considered?
5. **Provide use cases** - How would this feature be used?

### What makes a good feature request?

- **Clear problem statement** - Explain the problem you're trying to solve
- **Detailed solution** - Describe what you'd like to see implemented
- **Use cases** - Explain how this would be used and who would benefit
- **Priority level** - Help us understand how important this is to you

## ❓ Getting Support

For questions and support:

1. **Use the Question/Support template** - This helps us provide better assistance
2. **Search existing issues first** - Your question might already be answered
3. **Provide context** - Give us details about what you're trying to do
4. **Include code examples** - If relevant, show us what you've tried

## 📚 Documentation Issues

For documentation improvements:

1. **Use the Documentation template** - This helps us locate and fix issues
2. **Provide specific location** - Tell us exactly where the issue is
3. **Suggest improvements** - What should be changed or added?
4. **Explain the impact** - Why is this important?

## 🏷️ Labels and Triage

Our team uses labels to organize and prioritize issues:

- **Type labels**: `bug`, `enhancement`, `question`, `documentation`
- **Priority labels**: `high-priority`, `low-priority`
- **Status labels**: `needs-triage`, `in-progress`, `blocked`
- **Category labels**: `ui`, `performance`, `security`, etc.

## ⏱️ Response Times

We aim to:

- **Acknowledge new issues** within 2-3 business days
- **Provide initial response** to questions within 1 week
- **Triage bugs** based on severity and impact
- **Review feature requests** during our regular planning cycles

## 🚫 What Not to Report Here

Please don't use this repository for:

- **Security vulnerabilities** - Report these privately to security@vinkius.com
- **General support questions** - Use our community forums or documentation
- **Issues with third-party integrations** - Contact the respective maintainers first

## 🤝 Community Guidelines

- **Be patient** - Our team reviews issues regularly but may take time to respond
- **Be helpful** - Provide additional information when requested
- **Be respectful** - Treat everyone with respect and kindness
- **Stay on topic** - Keep discussions focused on the specific issue

## 📧 Contact

- **General issues**: Use this repository
- **Security issues**: security@vinkius.com
- **Community discussions**: [GitHub Discussions](https://github.com/vinkius-labs/vinkius-issues/discussions)
- **Documentation**: [docs.vinkius.com](https://docs.vinkius.com)

## 🙏 Thank You

Your contributions help make Vinkius better for everyone. Whether you're reporting a bug, requesting a feature, or helping others in the community, we appreciate your involvement!
